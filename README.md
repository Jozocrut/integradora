# Estructuras y permisos
- ´main´ Se establece como rama principal, no se permiten Pull Request(PR).
  No se podra realizar (PR) solo Desarrollador Frontend , Backend y SCM .
- ´develop´ Version estable del proyecto.
- ´feature´ Creada desde ´develop´ para el desarrollo de funcionalidades.
- ´hotfix´ Creada de ´main´ para la solucion de errores encontrados en produccion.

## Estructura de commit
Los ´commit´ se expresara una breve descripcion de lo que se halla modificado cada uno segun su rama
Ejemplo :´Implementada funcionalidad en formulario para agregar informacion´
